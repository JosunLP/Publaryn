# Code of Conduct

## Our standard

Publaryn is an open-source project for building secure package infrastructure. We want the project to be welcoming, constructive, and respectful for maintainers, contributors, and users.

Examples of behavior that help create a positive environment include:

- being respectful of different backgrounds, identities, experience levels, and viewpoints
- giving and receiving technical feedback with patience and professionalism
- assuming good intent while still being clear about problems and boundaries
- focusing discussion on the code, design, documentation, and user impact
- helping others learn without belittling them

Examples of unacceptable behavior include:

- harassment, intimidation, threats, or discriminatory language
- personal attacks, insults, trolling, or repeated bad-faith arguments
- publishing someone else's private information without permission
- sexualized language or unwelcome sexual attention
- deliberately disruptive behavior in issues, pull requests, reviews, or other project spaces

## Scope

This Code of Conduct applies to project spaces such as issues, pull requests, code reviews, discussions about repository changes, and other communication channels used to collaborate on Publaryn.

## Reporting

If you experience or witness unacceptable behavior, please contact the project maintainer for follow-up.

For sensitive concerns, use any private contact method that the maintainer has published on their GitHub profile. If no private contact method is available, open a minimal public issue requesting private follow-up without including sensitive personal details.

Security vulnerabilities should be reported through the process in [SECURITY.md](SECURITY.md), not through conduct reporting.

## Enforcement

Project maintainers may remove, edit, or reject comments, commits, code, issues, or pull requests that violate this Code of Conduct. They may also temporarily or permanently restrict participation when necessary to protect the community.

Enforcement decisions will be made with the goal of keeping the project safe, respectful, and productive.
