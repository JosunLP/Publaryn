# Support

This document explains where to ask questions and where to report problems for `Publaryn`.

## Where to go

### Security vulnerabilities

Do **not** open a public issue.

Use the private reporting path in [SECURITY.md](SECURITY.md): <https://github.com/JosunLP/Publaryn/security/advisories/new>

### Reproducible bugs or regressions

Open a GitHub issue using the bug report template and include:

- the affected area or ecosystem
- clear reproduction steps
- expected versus actual behavior
- logs, screenshots, or request/response details when relevant

### Feature requests or design ideas

Open a GitHub issue using the feature request template.

For major changes, please start with an issue before opening a pull request so the approach can be discussed first.

### Usage questions

Start by reading:

- [README.md](README.md)
- [CONTRIBUTING.md](CONTRIBUTING.md)
- [docs/concept.md](docs/concept.md)
- the relevant ADRs under [docs/adr/](docs/adr/)

If your question turns out to be a reproducible bug or a missing capability, please open the corresponding issue with as much context as possible.

## What to include when asking for help

Helpful reports usually include:

- your operating system and relevant tool versions
- the affected package ecosystem or API surface
- whether you are running locally or in a containerized deployment
- exact commands, requests, or configuration involved
- logs or screenshots that narrow down the problem

## Expectations

Publaryn is an actively developing open-source project. Support is best-effort and there is currently no guaranteed response SLA.
